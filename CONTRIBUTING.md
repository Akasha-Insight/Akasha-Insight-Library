# 🤝 贡献指南 (Contributing Guide)

感谢你有兴趣为 Akasha-Insight 添砖加瓦！正是因为有像你这样的贡献者，这个社区才能变得更强。

在开始之前，请花几分钟阅读以下准则。

## 🚫 安全红线 (Web3 特别版)

为了保障所有读者的资产安全，作为贡献者请务必遵守：
1.  **严禁夹带私货：** 禁止在教程或文章中植入带有推荐返佣 (Referral) 的链接，除非经过社区明确标记。
2.  **严查钓鱼链接：** 提交任何项目官网、工具链接前，请务必通过 DefiLlama 或 CoinGecko 核对 URL 的准确性。
3.  **代码安全：** 如果你分享的是代码脚本（如 Python 爬虫），请确保代码中不包含恶意的私钥上传逻辑。

## 🛠 我可以贡献什么？

* **纠错：** 发现文档里的错别字、失效的链接。
* **补充：** 完善现有的工具教程，补充新的数据指标解读。
* **原创：** 提交你自己撰写的深度研报或翻译优质的英文内容（需注明原作者授权）。
* **整理：** 发现好用的 Web3 工具，添加到资源列表。

## 📝 贡献流程 (Workflow)

如果你熟悉 GitHub，标准流程如下：

1.  **Fork 本仓库** 到你的个人账号。
2.  **Clone** 你的 Fork 版本到本地。
3.  创建一个新的分支：`git checkout -b my-new-content`
4.  进行修改或添加文件（请遵循下方的**内容规范**）。
5.  **Commit** 你的更改：`git commit -m 'Add: Dune SQL 基础教程'`
6.  **Push** 到你的远程仓库。
7.  回到本仓库页面，提交 **Pull Request (PR)**。

> **💡 不会用 Git？** > 你也可以直接在 GitHub 网页端找到你想修改的文件，点击右上角的 ✏️ 图标进行编辑，然后直接提交 PR。如果是上传新文章，建议先在社区 Discord 开票联系管理员。

## 📏 内容与格式规范

1.  **文件格式：** 所有文档必须使用 Markdown (`.md`) 格式。
2.  **命名规则：**
    * 文件名统一使用英文或拼音，单词间用连字符分隔。
    * ✅ 正确：`dune-analytics-guide.md`
    * ❌ 错误：`dune教程.md`, `Dune_Guide.md`
3.  **图片资源：**
    * 请将文章内的图片统一存放在 `/assets/images/` 目录下。
    * 引用路径使用相对路径：`![描述](../assets/images/pic.png)`。
4.  **排版：** 请善用二级标题、列表和代码块，保持排版整洁。

## ⏳ PR 审核机制

提交 PR 后，我们的维护团队会在 48 小时内进行 Review。
* 如果内容通过审核，我们会合并并在 Discord 公告中 @你。
* 如果有修改建议，我们会在 PR 评论区与你留言沟通。

再次感谢你的贡献！让我们一起构建最棒的 Web3 知识库。🚀
