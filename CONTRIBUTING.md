# 🤝 贡献指南 (Contributing Guide)

> **宁缺毋滥** — 这个仓库的每一条内容都应有它的分量。

感谢你想为 Akasha-Insight 添砖加瓦。在贡献之前，请理解我们的理念：

🚫 **我们拒绝**：随手丢个链接、AI 批量生成的垃圾内容、搬运工式的"伪贡献"
✅ **我们欢迎**：深度原创、精心整理、真正有价值的资源沉淀

## 什么内容值得提交？

### ✅ 高质量贡献

| 类型 | 合格标准 |
|------|---------|
| **教程** | 你自己写过/用过，有实操经验和截图 |
| **研报** | 有数据支撑、有独立观点、有风险提示 |
| **工具推荐** | 你**亲自用过至少一周**，能说出优缺点 |
| **翻译** | 获得原作者授权，附原文链接，专业术语准确 |

### ❌ 不接受

- 纯链接合集（没有说明为什么有用）
- AI 批量生成的内容（缺乏深度）
- 搬运其他平台的文章（除非原作者投稿）
- 未经验证的工具推荐
- 带返佣/推广链接的内容

## 贡献方式

### 方式一：先开 Issue 讨论（推荐）

不确定内容是否合适？先开个 Issue 聊聊：
1. 点 [New Issue](https://github.com/Akasha-Insight/Akasha-Insight-Library/issues/new)
2. 说明你想贡献什么内容
3. 达成一致后再动手写

### 方式二：提交 Pull Request

```bash
git clone https://github.com/Akasha-Insight/Akasha-Insight-Library.git
cd Akasha-Insight-Library
git checkout -b add/your-content-name
# 添加文件 → commit → push
git push origin add/your-content-name
# 在 GitHub 上创建 PR
```

## 📝 品质自检清单

提交前逐项确认：

- [ ] 内容有**原创成分**（你的经验、观点、整理）
- [ ] 不是纯 AI 输出（可以有 AI 辅助，但要有人类编辑的痕迹）
- [ ] 工具/资源链接你**亲自验证过**
- [ ] 翻译内容已获原作者授权
- [ ] 排版整洁，标准 Markdown
- [ ] 文件名英文连字符格式（如 `my-guide.md`）

加分项：
- [ ] 附上使用体验/踩坑经验
- [ ] 注明读者水平（新手/进阶）

---

## ⏳ 审核标准

| 维度 | 权重 | 说明 |
|------|------|------|
| **原创性** | ⭐⭐⭐⭐⭐ | 有独到的经验/见解，非搬运 |
| **实用性** | ⭐⭐⭐⭐ | 读者能立刻用上 |
| **准确性** | ⭐⭐⭐⭐ | 数据和链接准确无误 |
| **完整性** | ⭐⭐⭐ | 结构完整，有头有尾 |
| **排版** | ⭐⭐ | 清晰易读 |

> 只有 **至少 3 项达标** 的内容才会被合并。

## 🌟 贡献者积分体系

| 等级 | 积分 | 要求 | 权益 |
|------|------|------|------|
| 🥉 **贡献者** | 3分 | 3 篇优质内容被合并 | 列入贡献者名单 |
| 🥈 **核心贡献者** | 10分 | 持续输出高质量内容 | 优先审核通道 |
| 🥇 **维护者** | 20分 | 深度参与社区建设 | 直接合并权限 |

**积分计算：**
- 深度教程/研报（>1000字，有原创观点）→ **2分**
- 一般补充/整理/工具推荐 → **1分**
- 修正错别字/断链 → **0.5分**
- 纯链接提交 → **不计分**
